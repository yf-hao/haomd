## HaoMD 桌面图标替换与批量生成说明

> 适用场景：你有一张高分辨率 PNG（建议 1024×1024、方形），希望快速生成 Tauri 打包所需的多尺寸图标（`png` / `icns` / `ico`），并替换当前桌面图标。

---

## 一、Tauri 当前图标配置位置

**配置文件**：`app/src-tauri/tauri.conf.json`

```json
"bundle": {
  "active": true,
  "targets": "all",
  "icon": [
    "icons/32x32.png",
    "icons/128x128.png",
    "icons/128x128@2x.png",
    "icons/icon.icns",
    "icons/icon.ico"
  ]
}
```

**图标文件目录**：`app/src-tauri/icons/`

只要这个目录下对应路径存在正确的文件，并重新打包 Tauri 应用，桌面图标就会更新。

---

## 二、最简单方式：用在线工具一键生成

**前提**：准备好一张高分辨率 PNG（至少 512×512，推荐 1024×1024），背景透明或实色均可。

**步骤**：

1. 打开任意一个图标生成网站（任选其一）：
   - `https://www.favicon-generator.org`
   - `https://realfavicongenerator.net`
   - `https://appicon.co`
2. 上传这张 PNG，勾选桌面 App / PWA 等相关选项。
3. 生成完成后，下载压缩包，里面通常包含：
   - 多尺寸 PNG（16/32/64/128/256/...）
   - `icon.ico`（Windows 图标）
   - `.icns` 或相关 macOS 图标资源
4. 从压缩包里挑出需要的文件，拷贝到 `app/src-tauri/icons/` 目录，建议：
   - 用生成的 PNG 覆盖：
     - `32x32.png`
     - `128x128.png`
     - `128x128@2x.png`
   - 用生成的 `.icns` 文件覆盖：
     - `icon.icns`
   - 用生成的 `.ico` 文件覆盖：
     - `icon.ico`
5. 确认 `tauri.conf.json` 中的 `bundle.icon` 列表依然指向这些文件（路径不变即可）。
6. 在 `app` 目录重新打包应用（例如）：
   - `npm run tauri:build`
7. 安装新的构建产物，桌面 / Dock / Start 菜单中的图标就会更新。

> 说明：如果不想改配置，**最省事的方式是直接用新图标覆盖同名文件**，即保持文件名不变，只替换内容。

---

## 三、macOS 本地脚本化方案：从一张 PNG 批量生成

如果你希望在 macOS 上完全走本地命令行，可以使用系统自带的 `sips` 和 `iconutil` 来生成多尺寸 PNG 和 `.icns`，再用额外工具生成 `.ico`。

### 3.1 准备原始 PNG

1. 在 `app/src-tauri/icons/` 下放一张原始大图，例如命名为：
   - `app/src-tauri/icons/source.png`
2. 确保它是方形（宽高相等），尺寸至少为 512×512，推荐 1024×1024，以保证缩放后清晰度。

### 3.2 用 `sips` 生成多尺寸 PNG

> 以下命令只作为步骤说明，需在终端手动执行。

```bash
cd app/src-tauri/icons

# 从 source.png 生成 Tauri 配置中用到的尺寸
sips -z 32 32   source.png --out 32x32.png
sips -z 128 128 source.png --out 128x128.png
sips -z 256 256 source.png --out 128x128@2x.png
```

这样就更新了配置中用到的三个 PNG 文件：

- `32x32.png`
- `128x128.png`
- `128x128@2x.png`

### 3.3 生成 macOS `.icns` 图标

1. 在 `icons` 目录创建一个 `.iconset` 目录：

```bash
cd app/src-tauri/icons
mkdir HaoMD.iconset
```

2. 根据苹果图标命名规范，把刚生成的 PNG 拷贝进去（可以只用现有尺寸做一个精简版）：

```bash
cp 32x32.png      HaoMD.iconset/icon_16x16.png
cp 32x32.png      HaoMD.iconset/icon_16x16@2x.png
cp 32x32.png      HaoMD.iconset/icon_32x32.png
cp 128x128.png    HaoMD.iconset/icon_32x32@2x.png
cp 128x128.png    HaoMD.iconset/icon_128x128.png
cp 128x128@2x.png HaoMD.iconset/icon_128x128@2x.png
```

> 如果你后面需要更大的尺寸（256、512），可以再用 `sips` 生成对应 PNG，并按规则补进 `.iconset` 目录。

3. 使用 `iconutil` 从 `.iconset` 生成 `.icns`：

```bash
iconutil -c icns HaoMD.iconset -o icon.icns
```

完成后，`icons/icon.icns` 就会变成新图标。

### 3.4 生成 Windows `.ico` 图标

macOS 系统本身不直接提供生成 `.ico` 的工具，可以选择：

- **方式 A：继续使用在线工具**（推荐）
  1. 将 `source.png` 上传到在线图标生成网站；
  2. 只下载或提取生成的 `icon.ico` 文件；
  3. 覆盖 `app/src-tauri/icons/icon.ico`。

- **方式 B：安装 ImageMagick（可选）**

  1. 通过 Homebrew 安装（例如）：

     ```bash
     brew install imagemagick
     ```

  2. 在 `icons` 目录生成多尺寸 `.ico`：

     ```bash
     cd app/src-tauri/icons
     convert source.png -define icon:auto-resize=256,128,64,48,32,16 icon.ico
     ```

  3. 确认 `icon.ico` 已更新，`tauri.conf.json` 中仍然指向 `icons/icon.ico`。

---

## 四、更新图标后的打包与刷新注意事项

1. 确认 `tauri.conf.json` 中的 `bundle.icon` 路径与文件实际存在的路径一致。
2. 在 `app` 目录重新打包：
   - `npm run tauri:build`
3. macOS 上如果之前已经安装旧版本应用：
   - 建议先删除旧的 `.app` / 安装包，再安装新构建；
   - 如 Dock 或 Launchpad 仍显示旧图标，可以尝试：
     - 退出应用并从 Dock 移除，再从 Finder 重新拖入；
     - 或重启 Finder / 系统，等待图标缓存刷新。

> 小结：日常使用时，最简路径是：**准备一张统一的高分辨率 PNG → 用在线工具或少量命令生成多尺寸图标 → 覆盖 `app/src-tauri/icons` 下的同名文件 → 重新打包**。这样既不会改动 Tauri 配置，又能在所有平台看到一致的新图标。
