## 剪贴板图片粘贴到 Markdown 的实现方案

### 1. 背景与目标

在编辑器中按 `Ctrl+V` / `Cmd+V` 时，如果剪贴板中包含截图（图片），希望实现：

- 自动将图片保存到 **当前 Markdown 文件同目录下的某个子目录**（默认 `images/`）。
- 自动在光标位置插入一行 Markdown 图片语法：
  - `![](images/xxxx.png)`。
- 如果剪贴板中没有图片，则保留现有的文本粘贴行为。
- 方案需要 **解耦路径策略和底层实现**，方便未来按项目或用户配置自定义图片存储路径、命名规则等。

### 2. 总体架构

整体拆成三层，每层职责清晰、可替换：

1. **后端命令层（Tauri + Rust）**
   - 提供一个通用命令：
     - 负责从剪贴板读取图片，并保存到前端指定的目标目录。
     - 自动创建目标目录（包含多级目录）。
     - 返回实际生成的文件名（不含路径）。
   - 不关心当前 Markdown 文件路径、`images/` 等具体策略。

2. **前端路径策略层（TypeScript 模块）**
   - 根据“当前文件路径 + 工作区根路径 + 配置”计算：
     - 图片保存的 **绝对目录**：`targetDir`。
     - Markdown 引用使用的 **相对路径前缀**：例如 `images/`、`assets/img/2026/` 等。
   - 通过配置（代码内默认值 + 将来的设置 UI）决定目录结构与命名策略。
   - 可以在不改后端、不改编辑器的情况下，单独替换路径策略实现。

3. **编辑器粘贴监听层（EditorPane / CodeMirror）**
   - 监听编辑器 DOM 的 `paste` 事件。
   - 判断剪贴板中是否有图片：
     - 有图片：阻止默认粘贴，调用“路径策略 + 后端命令”，然后插入 `![](relPath)`。
     - 无图片：不处理，让现有文本粘贴继续工作。

数据流大致如下：

\[编辑器 paste 事件\] → \[路径策略 resolveImageTarget\] → \[Tauri: save_clipboard_image_to_dir\] → \[返回 file_name\] → \[前端拼接 relPath 并插入 Markdown\]

---

### 3. 后端命令层设计（Tauri + Rust）

#### 3.1 命令接口

新增一个通用命令，而不是把当前文件路径等逻辑写进后端：

```rust
#[tauri::command]
async fn save_clipboard_image_to_dir(
  target_dir: String,              // 目标目录（绝对路径），由前端决定
  suggested_name: Option<String>,  // 可选文件名前缀，如 "image"，用于生成最终文件名
) -> ResultPayload<ClipboardImageResult> {
  // 实现见下文
}
```

返回类型：

```rust
#[derive(Serialize)]
struct ClipboardImageResult {
  file_name: String, // 仅文件名，不含路径，如 "image-20260210-153045-ab12cd.png"
}
```

设计要点：

- **仅做一件事**：从剪贴板读图片 → 确保目录存在 → 生成文件名 → 写入文件 → 返回文件名。
- 不负责：
  - 计算当前文件所在目录；
  - 决定是不是 `images/`；
  - 生成 Markdown 内容。
- 这样该命令可以复用到其它场景（比如贴到 assets、临时目录等）。

#### 3.2 实现步骤

1. **规范化目标目录路径**：

```rust
let trace = new_trace_id();

let normalized_dir = match normalize_path(&target_dir) {
  Ok(p) => p,
  Err(e) => return ResultPayload::Err { error: e },
};
```

2. **确保目录存在（自动创建）**：

```rust
use std::fs;

if let Err(err) = fs::create_dir_all(&normalized_dir) {
  return err_payload(
    ErrorCode::IoError,
    format!("创建图片目录失败: {err}"),
    trace,
  );
}
```

3. **从剪贴板读取图片**（使用既有的 `arboard`）：

```rust
use arboard::Clipboard;

let mut cb = match Clipboard::new() {
  Ok(c) => c,
  Err(err) => {
    return err_payload(
      ErrorCode::IoError,
      format!("访问剪贴板失败: {err}"),
      trace,
    );
  }
};

let img = match cb.get_image() {
  Ok(img) => img, // 包含 bytes、width、height 等
  Err(err) => {
    return err_payload(
      ErrorCode::UNSUPPORTED,
      format!("剪贴板中没有图片或格式不支持: {err}"),
      trace,
    );
  }
};
```

4. **编码为 PNG 并写入文件**：

- 引入 `image` crate（如果尚未引入），用于将 `ImageData` 编码为 PNG。
- 文件名生成策略：
  - 前缀：`suggested_name.unwrap_or("image".into())`；
  - 后缀：当前时间（精确到毫秒）+ 短随机串；
  - 扩展名：`.png`。

示意：

```rust
use image::{ImageBuffer, Rgba};
use rand::{distributions::Alphanumeric, Rng};

let width = img.width as u32;
let height = img.height as u32;
let buffer: ImageBuffer<Rgba<u8>, _> =
  ImageBuffer::from_raw(width, height, img.bytes.into_owned())
    .ok_or_else(|| service_error(ErrorCode::UNSUPPORTED, "图片数据无效", Some(trace.clone())))?;

let prefix = suggested_name.unwrap_or_else(|| "image".to_string());
let rand_suffix: String = rand::thread_rng()
  .sample_iter(&Alphanumeric)
  .take(6)
  .map(char::from)
  .collect();
let timestamp = chrono::Local::now().format("%Y%m%d-%H%M%S-%3f");
let file_name = format!("{}-{}-{}.png", prefix, timestamp, rand_suffix);

let full_path = normalized_dir.join(&file_name);
if let Err(err) = buffer.save(&full_path) {
  return err_payload(
    ErrorCode::IoError,
    format!("写入图片失败: {err}"),
    trace,
  );
}

ok(ClipboardImageResult { file_name }, trace)
```

> 注：这里使用 `normalize_path`、`err_payload`、`ok` 等工具函数，和现有后端保持一致风格。

5. **注册命令**：

- 在 `run()` 中的 `invoke_handler` 里加入：`save_clipboard_image_to_dir`。

### 4. 前端路径策略层设计

目标：提供一个可配置的“计算图片存储位置”的函数，将来可以在不动后端与编辑器的前提下，支持多种路径策略。

#### 4.1 配置结构

在前端新建模块，例如：`app/src/modules/images/imagePasteStrategy.ts`：

```ts
export type ImagePathStrategyMode = 'current-file-dir' | 'workspace-root'

export interface ImagePathStrategyConfig {
  mode: ImagePathStrategyMode
  subdir: string          // 默认 'images'
  filenamePrefix: string  // 默认 'image'
}

export interface ResolvedImageTarget {
  targetDir: string  // 绝对目录，用于 Tauri 命令
  relDir: string     // 相对目录前缀，用于 Markdown 中（如 'images' 或 'assets/img'）
}
```

**默认配置**：

```ts
export function loadDefaultImagePathStrategyConfig(): ImagePathStrategyConfig {
  return {
    mode: 'current-file-dir',
    subdir: 'images',
    filenamePrefix: 'image',
  }
}
```

将来可以扩展为：

- 从本地 JSON / Store / 设置面板加载；
- 支持模板字符串（例如 `images/{yyyy}/{MM}`、`images/{fileName}` 等）。

#### 4.2 目标路径解析函数

```ts
import path from 'path-browserify' // 或 Node 风格 polyfill

export function resolveImageTarget(
  currentFilePath: string,
  workspaceRoot: string | null,
  cfg: ImagePathStrategyConfig,
): ResolvedImageTarget {
  const baseDir = cfg.mode === 'workspace-root'
    ? workspaceRoot ?? path.dirname(currentFilePath)
    : path.dirname(currentFilePath)

  const targetDir = path.join(baseDir, cfg.subdir)
  const relDir = cfg.subdir // Markdown 中使用相对路径，例如 'images'

  return { targetDir, relDir }
}
```

> 说明：
> - 当前版本中，`relDir` 等于 `subdir`，即在 Markdown 中使用相对当前文件的路径 `images/xxx.png`。
> - 如果未来需要更复杂的相对路径（例如相对工作区根），可以在这里单独处理。

### 5. 编辑器粘贴监听层设计

在 `EditorPane`（或负责 CodeMirror 的组件）中：

#### 5.1 需要的输入信息

组件需要额外从 `WorkspaceShell` 传入：

- `currentFilePath: string | null`：当前 Tab 对应文件的绝对路径（未保存时为 null）。
- `workspaceRoot: string | null`：当前工作区根路径（可选，可从侧边栏根或配置中获取）。
- 一个状态提示回调（已有）：`onStatusMessage?: (msg: string) => void`。

#### 5.2 绑定 paste 事件

在 `useEffect` 中使用 `editorViewRef.current.dom` 绑定 / 解绑 `paste`：

```ts
useEffect(() => {
  const view = editorViewRef.current
  if (!view) return

  const handlePaste = async (e: ClipboardEvent) => {
    const dt = e.clipboardData
    if (!dt) return

    const hasImage = Array.from(dt.items).some(item => item.type.startsWith('image/'))
    if (!hasImage) return // 保持原有文本粘贴行为

    e.preventDefault()
    e.stopPropagation()

    if (!currentFilePath) {
      onStatusMessage?.('当前文件尚未保存，无法确定图片存放目录')
      return
    }

    const cfg = loadDefaultImagePathStrategyConfig()
    const { targetDir, relDir } = resolveImageTarget(currentFilePath, workspaceRoot, cfg)

    try {
      const result = await invoke('save_clipboard_image_to_dir', {
        targetDir,
        suggestedName: cfg.filenamePrefix,
      }) as { data?: { file_name: string } }

      const fileName = result?.data?.file_name
      if (!fileName) {
        onStatusMessage?.('粘贴图片失败：后端未返回文件名')
        return
      }

      const relPath = `${relDir}/${fileName}`
      insertTextAtCursor(`![](${relPath})`)
    } catch (err) {
      onStatusMessage?.(`粘贴图片失败：${String(err)}`)
    }
  }

  view.dom.addEventListener('paste', handlePaste as any)
  return () => {
    view.dom.removeEventListener('paste', handlePaste as any)
  }
}, [editorViewRef, currentFilePath, workspaceRoot])
```

> 注意：
> - 只有在检测到剪贴板中确实有图片时才 `preventDefault`，确保不影响普通文本粘贴。
> - `insertTextAtCursor` 使用 CodeMirror 的 `dispatch` 实现，复用现有的插入逻辑。

#### 5.3 insertTextAtCursor 辅助函数

```ts
function insertTextAtCursor(text: string) {
  const view = editorViewRef.current
  if (!view) return
  const { state } = view
  const { from, to } = state.selection.main
  view.dispatch(state.update({
    changes: { from, to, insert: text },
    selection: { anchor: from + text.length },
    scrollIntoView: true,
  }))
}
```

### 6. 未来扩展点

1. **可配置路径策略**：
   - 在设置面板中允许用户选择：
     - 基准目录：当前文件所在目录 / 工作区根目录；
     - 子目录：`images` / `assets/img` / 自定义；
     - 是否按日期、按文件名分组（例如 `images/2026/02/` 或 `images/{fileName}/`）。
   - 对应实现：扩展 `ImagePathStrategyConfig` 和 `resolveImageTarget` 即可。

2. **文件名模板**：
   - 支持模式：`image-{yyyyMMdd}-{HHmmss}-{rand4}.png`；
   - 在后端根据传入的模板生成文件名。

3. **Markdown 片段自定义**：
   - 默认插入 `![](<path>)`；
   - 将来可以允许用户配置插入片段，如：
     - `![alt text](<path>)`；
     - 或带标题 / 带高度宽度的 HTML `<img>`。

4. **多图片粘贴支持**：
   - 如果剪贴板中有多张图片（部分系统支持），可循环处理，每张插入一行 `![](...)`。

5. **与原生菜单 Paste 的整合**：
   - 当前方案针对编辑器 DOM 的快捷键粘贴；
   - 将来可以在处理 `native://paste` 事件时也复用该逻辑，实现菜单粘贴同样支持图片。
